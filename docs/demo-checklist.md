# デモ・動作確認チェックリスト

このチェックリストは、GUI/CUIクライアント、サーバ、再戦、切断処理、順位表示が発表デモで問題なく動くことを確認するためのもの。

## 起動前確認

- [ ] Java 21 が使われている
  - `java -version`
  - `mvn -version`
- [ ] コンパイルが成功する
  - `mvn clean compile`
  - または `make build`
- [ ] サーバが起動できる
  - `make server`
- [ ] GUIクライアントが起動できる
  - `make gui-client name=Alice`
- [ ] CUIクライアントが起動できる
  - `make client name=Alice`

## 接続・ロビー確認

- [ ] 1人目が接続できる
- [ ] 2人目が接続できる
- [ ] GUI上に自分の `PlayerName` が表示される
- [ ] GUI上に `Status: WAITING` が表示される
- [ ] ロビー人数アイコンが参加人数に応じて増える
- [ ] Ready前の参加者が未Ready表示になる
- [ ] Readyボタンを押すと自分の状態が `READY` になる
- [ ] Ready人数が `1/2`, `2/2` のように更新される
- [ ] 1人だけReadyではゲームが始まらない
- [ ] 全員Readyでゲームが始まる

## 満員・人数制限確認

- [ ] 4人まで接続できる
- [ ] 5人目は接続拒否される
- [ ] 5人目のGUIまたはCUIに接続失敗理由が表示される
- [ ] 5人目拒否後も既存4人のゲームは続行できる

## 出題確認

- [ ] ゲーム開始後、問題文が表示される
- [ ] 問題文が1文字ずつ逐次表示される
- [ ] 選択肢4つがボタンとして表示される
- [ ] 回答ボタンが押せる
- [ ] 問題文や選択肢が画面から大きくはみ出さない
- [ ] `GameConfig.QUESTIONS_PER_GAME` で指定した問題数でゲームが終了する
- [ ] 毎回問題順がランダムに変わる
- [ ] 1ゲーム中に同じ問題が重複して出ない

## 回答・判定確認

- [ ] 正解ボタンを押すと、そのプレイヤーが正解者として表示される
- [ ] 正解者に1点加算される
- [ ] 不正解ボタンを押すと、不正解ログが表示される
- [ ] 不正解後、同じプレイヤーの連続誤答が不自然に加算されない
- [ ] 全員不正解の場合、正解者なしで次の問題へ進む
- [ ] 早押し時、先にサーバで処理された回答だけが勝つ
- [ ] 正解後、遅れて押した回答が勝者扱いにならない

## スコア確認

- [ ] 各ラウンド後にスコアが更新される
- [ ] 自分のスコアがGUIログで分かる
- [ ] CUIでもスコアが表示される
- [ ] ゲーム終了時点のスコアと表彰台のスコアが一致する

## 決着画面確認

- [ ] ゲーム終了後、決着画面に遷移する
- [ ] 1位〜3位は表彰台に表示される
- [ ] 4位は表彰台に乗らない
- [ ] 4位用の不要な地面表示が出ない
- [ ] 各プレイヤーの `PlayerName` が表示される
- [ ] 各プレイヤーのポイントが表示される
- [ ] 長い名前でも表示が大きく崩れない
- [ ] `playerId` は表示されない

## 同率順位確認

順位は「そのプレイヤーより高いスコアの人数 + 1」で決める。

- [ ] 全員同点なら全員1位になる
- [ ] 1位が2人同点の場合、両方1位になる
- [ ] 1位が2人同点の場合、次の順位が3位になる
- [ ] 2位が2人同点の場合、両方2位になる
- [ ] 2位が2人同点の場合、次の順位が4位になる
- [ ] 3位が2人同点の場合、両方3位になる
- [ ] 4位のプレイヤーだけ表彰台外に表示される
- [ ] 「4人いるから必ず誰かが4位」ではなく、順位計算に従って表示される

## 再戦確認

- [ ] 決着画面で `もう一度対戦する` を押せる
- [ ] 押すとロビー画面に戻る
- [ ] 押したプレイヤーはReady状態になる
- [ ] 全員が再戦Readyになるまで次ゲームは始まらない
- [ ] 全員Readyで次ゲームが始まる
- [ ] 再戦時、スコアが0から始まる
- [ ] 再戦時、問題順が再シャッフルされる
- [ ] 再戦後も `GameConfig.QUESTIONS_PER_GAME` で指定した問題数で終了する

## 退出・切断確認

- [ ] `部屋を抜ける` でGUIクライアントが終了する
- [ ] CUIで `d` / `quit` / `exit` で切断できる
- [ ] クライアントが途中退出してもサーバが落ちない
- [ ] サーバを `Ctrl+C` で終了するとGUI側が切断状態になる
- [ ] サーバを `Ctrl+C` で終了するとCUI側も終了する
- [ ] サーバ未起動でGUIを起動した場合、分かりやすいエラーが出る

## CUI確認

- [ ] `ready` / `r` / `準備` が使える
- [ ] 数字入力で回答できる
- [ ] 範囲外の数字入力に警告が出る
- [ ] `status` / `s` / `状態` が使える
- [ ] `h` / `help` / `ヘルプ` が使える
- [ ] ゲーム終了時にCUIが終了する

## デモ本番確認

- [ ] デモ用の起動コマンドをREADMEで確認できる
- [ ] 2人プレイのデモ手順がある
- [ ] 4人プレイのデモ手順がある
- [ ] 失敗時の復旧手順がある
  - 例: `Ctrl+C` -> `mvn clean compile` -> `make server`
- [ ] 発表用PCの画面サイズでGUIが見切れない
- [ ] プロジェクター表示でも文字が読める

## 最低限の本番前チェック

- [ ] 2人GUIで最後まで遊べる
- [ ] 指定問題数で終了する
- [ ] 表彰台の順位とスコアが正しい
- [ ] 再戦できる
- [ ] サーバを落としてもクライアントが固まらない
