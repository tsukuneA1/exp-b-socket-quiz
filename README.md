# ソフトウェア制作 グループ制作プロジェクト概要

## 課題の目的

このプロジェクトの目的は、Javaによるネットワークプログラミングを実際に体験し、TCP/IP、IPアドレス、ホスト名、ポート番号、ソケット通信の基礎を理解すること。

単にプログラムを完成させるだけでなく、グループで計画を立て、ゴールを設定し、役割分担を行い、進捗を報告し、最終的に発表資料としてまとめることも目的に含まれる。

大学側は、以下の能力を確認しようとしていると考えられる。

- Javaでsocketを用いたネットワーク通信を実装できること
- 自分たちで制作するプログラムのゴールを設定できること
- グループ内で役割分担し、計画的に開発できること
- 作成したプログラムの仕組みや工夫点を説明できること
- 他グループの発表と比較して、自分たちの成果を評価できること

## 制作するプログラムの条件

制作するプログラムは、socketを使用してネットワーク通信を行うものであれば形式は自由。

例としては以下のような形態が想定されている。

- サーバ・クライアント型
  - Web風アプリケーション
  - データ配信
  - チャットなど
- P2P型
  - チャット
  - 対戦ゲームなど
- 分散並列処理型
  - 画像処理
  - 科学技術計算など

## 技術的な制約

- 使用言語は Java。
- Javaのsocketインタフェースを用いる。
- ネットワーク通信を行うプログラムであること。
- 最終発表では、自分のPC上で複数のプログラムを同時に動作させる。
- デモではローカルのTCP/IP機能を使い、ホスト名は `localhost` を使用する。
- 実装したプログラムのソースコードを提出する必要がある。

## グループワーク上の要件

- グループで制作する課題の概要を決める。
- ゴールまでのスケジュールを立てる。
- 各メンバーの役割分担を明確にする。
- 中間発表では、進捗状況、作成したクラス、今後の予定を説明する。
- 最終発表では、プロジェクト概要、プログラムの説明、デモを行う。
- 最終レポートでは、自分の担当部分と、他グループとの比較評価を書く。

## 主な提出物

### 1. 中間発表スライド

- グループごとに提出。
- 形式は `ppt` ファイル。
- 代表者がMoodleにアップロードする。
- 表紙にグループ名とメンバーリストを書く。
- 各自の役割分担を明記する。
- 内容には以下を含める。
  - その時点で作成したクラスの説明
  - 進捗状況
  - 今後の予定

### 2. 最終発表スライド

- グループごとに提出。
- `ppt` を `pdf` に変換して提出する。
- 表紙にグループ名とメンバーリストを書く。
- 発表では、プロジェクト概要とプログラムの説明をメンバー全員で行う。
- デモを含める。

### 3. ソースコード

- グループごとに提出。
- 各自が作成したソースコードや必要なデータを `zip` にまとめる。
- ファイル名はグループ名にする。
- 代表者が提出する。

### 4. 最終レポート

- 各自が個別に提出する。
- 形式はPDF。
- 内容には以下を含める。
  - グループメンバーの名前と学籍番号
  - 作成したプログラムの概要
  - プログラムの動作説明
  - 動作していることを示すスクリーンショット
  - 工夫した点
  - 自分の役割と担当部分
  - 他グループの発表を最低2つ評価
  - 他グループと比較した自分たちの課題の自己評価

## マイルストーン

### Milestone 1: 企画決定

- 作るネットワークアプリケーションのテーマを決める。
- サーバ・クライアント型、P2P型、分散処理型など、全体構成を決める。
- 最終発表で見せるデモ内容を決める。
- 各メンバーの担当範囲を決める。

### Milestone 2: 設計

- 必要なクラスを洗い出す。
- クライアント側とサーバ側の責務を分ける。
- 通信内容、メッセージ形式、ポート番号、実行方法を決める。
- 最低限動くバージョンの仕様を決める。

### Milestone 3: 最小実装

- サーバとクライアントが接続できる状態にする。
- `localhost` 上で通信できることを確認する。
- 最低限のデータ送受信を実装する。
- エラー時の挙動を確認する。

### Milestone 4: 機能実装

- 企画した主要機能を実装する。
- 複数クライアント対応、画面表示、ゲームロジック、チャット機能など、プロジェクト固有の処理を追加する。
- 各メンバーの担当部分を統合する。

### Milestone 5: 中間発表準備

- 現時点でできている機能を整理する。
- 作成したクラスを説明できるようにする。
- 進捗状況と今後の予定をスライドにまとめる。
- 役割分担を明記する。

### Milestone 6: デモ完成

- 最終発表で見せる操作手順を決める。
- ローカル環境で複数プログラムを同時に起動して動作確認する。
- `localhost` を用いたTCP/IP通信でデモできるようにする。
- 発表中に失敗しにくいよう、実行手順を簡単にする。

### Milestone 7: 最終提出準備

- 発表スライドをPDF化する。
- ソースコードと必要なデータをZIPにまとめる。
- 不要な `.class` ファイルや一時ファイルを除外する。
- READMEに実行方法を書く。
- メンバー全員の名前、学籍番号、役割分担が資料に入っているか確認する。

### Milestone 8: 個人レポート作成

- 自分の担当部分を具体的に説明する。
- プログラムが動作しているスクリーンショットを入れる。
- 工夫した点、難しかった点を書く。
- 他グループの発表を最低2つ比較評価する。
- 自分たちの課題の良かった点・悪かった点を整理する。

## READMEに書いておくべき実行情報

最終的なREADMEには、少なくとも以下を書く。

- プロジェクト名
- 概要
- メンバーと担当
- 使用技術
- 必要なJavaバージョン
- コンパイル方法
- 実行方法
- サーバとクライアントの起動順
- 使用するポート番号
- デモ手順
- 既知の制限
